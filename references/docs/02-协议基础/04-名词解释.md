# 名词解释

## 充电卡

**卡类型**: 平台默认充电卡为M1卡(不带储值)

**物理卡号**:
- 读卡器读取到的卡号
- 卡号储存在第0扇区0块
- 用于充电桩与服务器数据交互

**逻辑卡号**:
- 卡面印刷的卡号
- 用于显示在桩屏幕上便于用户核对卡信息

**其他卡类型**: IC卡或者CPU卡均采用平台M1卡的鉴权模式,不使用数据储存与写入功能

## 交易流水号

**定义**: 交易流水号为一次充电操作过程的统一标记,从远程启动充电或者卡鉴权回复时产生到最后桩结束充电的交易记录均使用同一个流水号。

**生成规则**:
- 格式: 桩号(7bytes) + 枪号(1byte) + 年月日时分秒(6bytes) + 自增序号(2bytes)
- 由平台产生(离线模式由桩按此规则生成)

**示例**: 32010600019236012001061803423060

## 计损比例

**定义**: 计损比例定义在费率帧中,此项非零时,充电桩需要对上送平台充电量加上此比例。

**应用场景**:
- 实时监测数据中的"计损电度" = "电度"基础上加上此比例
- 交易记录中的"计损尖电量"、"计损峰电量"、"计损平电量"、"计损谷电量"、"计损总电量"

**注意**: 目前平台不支持计损功能,计损比例置0

## 桩编号

**定义**: 由平台生成提供给桩使用,不支持桩企自行生成的桩编号

**示例**: 3201060001923601

## 枪号

**定义**: 由7个字节桩编号 + 1个字节的枪号生成

**示例**:
- 1号枪: 3201060001923601
- 2号枪: 3201060001923602
